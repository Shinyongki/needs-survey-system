# 04. 통합 검증

> 트랙 A·B·C 모두 완료 후 실행  
> 전체 연결 상태 확인 및 버그 수정

---

## 검증 순서

### 1단계 — 서버 기동 확인

```bash
# 터미널 1
cd C:\Projects\설문시스템\server
node index.js
# → "서버 실행: http://localhost:3001" 출력 확인

# 터미널 2
cd C:\Projects\설문시스템\client
npm run dev
# → "http://localhost:5173" 출력 확인
```

---

### 2단계 — API 연결 확인

아래 항목을 순서대로 확인:

| 확인 항목 | 방법 | 기대 결과 |
|---|---|---|
| DB 초기화 | `node server/db/init.js` 재실행 | 오류 없음, "이미 존재" 메시지 |
| 기관 목록 | 브라우저에서 기관 관리 화면 접속 | 59개 기관 표시 |
| 지자체 필터 | 창원시 선택 | 10개 기관만 표시 |
| 규모 필터 | 대형 선택 | 대상자 1000명 이상 기관만 표시 |
| 응답 현황 | 응답 현황 화면 접속 | 매트릭스 표시, 전부 — 상태 |

---

### 3단계 — CSV 업로드 테스트

아래 테스트용 CSV를 사용하여 업로드 기능 확인:

```csv
타임스탬프,기관선택,Q1,Q2,Q3,Q4,Q5
2026-04-01 09:00:00,A48310001_거제노인통합지원센터,4,3,5,4,3
2026-04-01 09:05:00,A48310002_거제사랑노인복지센터,3,4,4,5,3
2026-04-01 09:10:00,A48880002_거창노인통합지원센터,5,5,4,3,4
```

확인 항목:
- 파일 선택 후 미리보기: 총 3행, 기관 매칭 3개
- [DB에 저장] 클릭 후: 삽입 15건 (3기관 × 5문항)
- 응답 현황 화면 재조회: 해당 기관 셀이 ✅로 변경

---

### 4단계 — 버그 수정 체크리스트

아래 항목 중 문제 발생 시 즉시 수정:

- [ ] 응답 현황 매트릭스 데이터 누락
- [ ] 기관 수정 저장 후 목록 미갱신

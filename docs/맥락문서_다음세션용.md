### Phase 2. 개별 사업 계획안·결과보고서 작성

현재 계획안·결과보고서는 **광역 전체 통합 문서**만 존재합니다. 개별 사업별 문서가 없습니다.

#### 2-1. 개별 사업 계획안 구조 (16종 + 사전 수요조사)

각 사업별 계획안은 다음 구조를 따릅니다:

```
1. 사업 개요
   - 사업명, 목적, 대상, 시기
2. 전년도 결과 요약 (2027년 이후)
   - 현장설문 만족도 주요 결과
   - 근거 문장
3. 올해 추진방향
   - 근거 문장 형식으로 명시
4. 세부 운영 계획
   - 일정, 방법, 담당자
5. 평가 방법
   - 현장설문 활용 계획
```

**작성 대상 (우선순위 순):**
1. J01 수행기관 모니터링·점검
2. J07 실무협의회 (사전 수요조사 포함)
3. J03·J04 역량강화교육
4. J05·J06 심리지원교육
5. J08 컨설팅 지원
6. J09 특화지원서비스·위기사례 연계
7. J10 퇴원환자 단기집중서비스
8. J11 노무 상담 지원
9. J12 ICT 지원사업
10. J13 나눔사업
11. J14 안심 돌봄 캠페인
12. J15 홍보사업
13. J16 유공자 표창지원사업
14. J17 종사자 지원사업

#### 2-2. 개별 사업 결과보고서 구조

```
1. 사업 실적
   - 시행 횟수, 참여 기관·인원
2. 현장설문 결과
   - 공통 문항 만족도 (CSI)
   - 특화 문항 주요 결과
   - 권역별·연령별·경력별 비교
3. 현장메모 주요 내용
4. 근거 문장
   - "○○% 만족 → 동일 방식 유지"
   - "○○% 불만족 → ○○ 개선"
5. 다음 회차 개선사항
```

### Phase 3. 시스템 개선 (Claude Code)

**모든 문서 작업(Phase 1·2) 완료 후 Claude Code로 전달.**

#### 3-1. 설문 구조 변경 반영

| 항목 | 현재 | 변경 |
|---|---|---|
| S01 question_count | 55 또는 56 | **61** |
| S02 question_count | 57 또는 59 | **65** |
| J04·J06 question_count | 13 | **14** |
| CSV 파싱 — 기관명 열 | 있음 | **삭제** (기관명 열 없음) |
| CSV 파싱 — 권역 열 | 없음 | **추가** (도입-3 위치) |
| 분석 단위 | 기관별 | **권역·연령대·경력별** |

#### 3-2. 권역별 분석 기능 추가

- 권역 코드: 북부(1)·서부(2)·중부(3)·동부(4)
- 권역별 집계·비교 기능
- 기존 기관별 분석 → 권역별 분석으로 전환

#### 3-3. 현장설문 만족도 분석 구조 변경

- 기존: 만족도 + 욕구조사 혼합 분석
- 변경: 만족도·경험 평가 중심 분석 (J04·J06만 욕구 포함)
- CSI 산출 방식 유지

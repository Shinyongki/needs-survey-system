# 노인맞춤돌봄서비스 광역지원기관 — 현황분석·욕구조사 설문체계 종합 정리

> 경상남도사회서비스원 광역지원기관
> 작성일: 2026년 3월
> 이 문서는 프로젝트 전체 구조를 7개 섹션으로 요약한 참조 문서입니다.

## 1. 설문 체계 구성

### 설문 종류 및 기본 정보

| 설문 | 시기 | 문항 수 | 응답 대상 | 비고 |
|---|---|---|---|---|
| **상반기 통합설문** | 4월 | 55문항 | 전담사회복지사 | 욕구 40 + 현황 15 |
| **현장설문** | 각 사업 종료·회차 직후 | 10문항 × 16종 | 전담사회복지사 14종 / 생활지원사 2종 | 만족도 5 + 욕구 5 |
| **하반기 통합설문** | 11~12월 | 57문항 | 전담사회복지사 | 욕구 42 + 현황 15 |

> **명칭 확정:** "사업별 만족도 설문" → **"현장설문"**
> - 만족도만이 아닌 욕구조사를 포함하는 구성
> - 회차 종료 직후 현장에서 바로 작성하는 성격 반영
> - 실무협의회처럼 연간 다회 진행 사업은 회차별로 등록 (예: 실무협의회 1회차 현장설문)

### 현장설문 응답자 구분

| 직책 | 응답 설문 | 종수 |
|---|---|---|
| 전담사회복지사 | 상반기·하반기 통합설문 + J01·J03·J05·J07~J17 | 통합 2종 + 현장 14종 |
| 생활지원사 | J04(역량강화교육) · J06(심리지원교육) | 현장 2종 |

## 2. 현황분석과 욕구조사의 구분

각 설문 안에 성격이 다른 두 종류의 문항이 혼재합니다.

### 현황분석 문항 — "지금 어떤가"

- **위치:** 통합설문 각 15문항 (현황분석 보완 문항)
- **역할:** 응답자의 주관적 체감을 수치화. 시스템 데이터만으로 파악하기 어려운 현장 감각 보완
- **출처:** 설문 현황체감 문항 + 시스템 데이터 + 결과보고서 + 현장메모
- **활용:** 수치 → 시사점 형태로 현황분석 작성

### 욕구조사 문항 — "무엇을 원하는가"

- **위치:** 상반기 40문항 / 하반기 42문항 / 현장설문 특화 5문항
- **역할:** 응답 비율(%)이 근거 문장의 핵심 재료
- **활용:** ○○% 요청 → 근거 문장 생성 → 추진방향 수립

> **핵심 구분**
> - 현황분석은 설문 외에도 시스템 데이터·결과보고서·현장메모가 주재료
> - 욕구조사는 설문 없이는 성립 불가

## 3. 추진방향의 위치

추진방향은 설문이 직접 제시하지 않습니다.
현황분석 결과와 욕구조사 결과를 입력값으로 받아 **담당자가 판단·작성**합니다.

### 근거 문장 형식

```
욕구조사 근거:
"○○조사 결과 ○○%가 ○○을 요청하여 ○○을 추진한다."

현황분석 근거:
"전년도 ○○ 결과 ○○이 확인되어 ○○을 추진한다."

결합 형식:
"○○ 현황체감 결과 ○○%가 ○○한 것으로 나타나고,
 ○○조사 결과 ○○%가 ○○을 요청하여 ○○을 추진한다."
```

### 생성 구조

```
현황분석 결과 (수치 + 시사점)
        +
욕구조사 결과 (응답 비율 %)
        ↓
담당자 판단·작성
        ↓
추진방향 확정 → 계획안·결과보고서 수록
```

## 4. 전체 운영 흐름 — 4단계

### 단계별 구조

```
1단계  설문 실시
       → 응답 수집 → DB 저장 → CSV 내보내기

2단계  분석 실행 (별도 Claude 세션)
       → CSV + 분석 지시어 붙여넣기
       → 분석결과보고서 생성
         (기술통계 · 갭분석 · IPA · 텍스트 · 근거문장 A·B·C형)

3단계  결과 정리 (이 프로젝트 세션)
       → 분석결과보고서 업로드
       → 현황분석 · 욕구조사 · 추진방향 초안 정리
       → 담당자 검토·수정·확정

4단계  문서 수록
       → 계획안 · 결과보고서 해당 섹션에 수록
```

### 핵심 구분

| 단계 | 입력값 | 출력물 |
|---|---|---|
| 2단계 | **CSV 원천 데이터** + 분석 지시어 | 분석결과보고서 |
| 3단계 | **분석결과보고서** | 현황분석·욕구조사·추진방향 초안 |

> ⚠️ 2단계 입력(CSV)과 3단계 입력(보고서) 혼용 금지

### 문서별 투입 설문

| 문서 | 핵심 투입 | 보완 투입 |
|---|---|---|
| **계획안** | 하반기 통합설문 (핵심) | 현장설문 누적 · 상반기 통합설문 (참고) |
| **결과보고서** | 현장설문 (즉시 반영) | — |

## 5. 분석 지시어 체계

세 설문 모두 분석 지시어 완성.

| 설문 | 지시어 | 주요 분석 모델 |
|---|---|---|
| **상반기 통합설문** | 유형 1~8 | 현황집계 · 욕구순위 · 교차분석 · 근거문장 · 시각화 · 섹션집중 · 보고서 |
| **하반기 통합설문** | 유형 1~9 | 상반기 동일 + **상반기↔하반기 비교** |
| **현장설문 16종** | M1~M8 × 16종 | 기술통계 · CSI · 갭 · IPA · 욕구순위 · 텍스트 · **근거문장(A·B·C형)** |

### 현장설문 분석 모델 (M1~M8) 개요

| 모델 | 내용 |
|---|---|
| M1 | 기술통계 — 빈도·비율·평균 |
| M2 | 만족도지수(CSI) — 100점 환산·순위 |
| M3 | 균형 프로파일 — 레이더 차트 |
| M4 | 갭(Gap) 분석 — 단계별 병목 진단 |
| M5 | IPA 분석 — 중요도×만족도 4분면 |
| M6 | 욕구순위 분석 — Top 3·조합 패턴 |
| M7 | 텍스트 분석 — 키워드·유형 분류 |
| M8 | 근거문장 생성 — A·B·C형 3가지 |

### 상반기↔하반기 이행 확인 관계

현장설문 분석 시 상반기 통합설문과의 연결이 필요합니다.

```
상반기 욕구조사 (○○% 요청)
        ↓ 사업 실행
현장설문 (해당 항목 만족도·욕구 측정)
        ↓ 이행 확인 분석
"상반기 욕구조사 결과 ○○%가 ○○을 요청하여 반영하였으며,
 현장설문 결과 ○○점으로 나타났다."
```

> 이행 확인 분석이 현장설문 분석 지시어에 포함되어야 완전한 근거 생산 가능

## 6. 이 프로젝트의 역할

### 저장된 내용

| 구분 | 내용 |
|---|---|
| 분석 지시어 | 상반기 통합설문(유형 1~8) · 하반기(유형 1~9) · 현장설문 M1~M8 × 16종 |
| 설문 문항 전문 | S01 55문항 · S02 57문항 · 현장설문 16종 |
| 문서 양식 | 계획안 · 통합결과보고서 |
| 프로젝트 맥락 | 전체 설계·분석·설계 논의 이력 |

### 활용 방법 (3단계)

1. **분석결과보고서 업로드** — CSV 분석 후 생성된 보고서를 이 대화에 첨부
2. **자연어로 요청** — 맥락 설명 없이 바로 실행 가능
3. **결과 수령** — 현황분석·욕구조사·추진방향 초안 즉시 출력

> ※ 위 명령어는 패턴 예시이며, 자연어 요청도 작동합니다.
> ※ 분석(2단계)과 정리(3단계)의 입력값을 혼동하지 않도록 주의.

## 7. 시스템 구현 방향 (Claude Code)

분석 계산은 시스템이 직접 하지 않습니다.
시스템의 핵심 기능은 **관리·저장·출력**입니다.

### 화면 구성

| 화면 | 기능 |
|---|---|
| **설문 관리** | 연간 일정 · 회차 등록 · 응답 현황 · CSV 내보내기 · 설문 링크 배포 |
| **분석 지시어** | 설문·사업·분석유형 선택 → 지시어 자동 생성 → 복사 |
| **분석 결과 입력** | 현황분석·욕구조사·추진방향 초안 저장·수정·확정 |
| **문서 출력** | 계획안·결과보고서 섹션 생성 · Word·PDF 출력 · 환류 이력 관리 |

### 핵심 DB 테이블

| 테이블 | 내용 |
|---|---|
| `survey_sessions` | 설문 회차 정보 (사업명·시기·횟차·응답 목표 수) |
| `analysis_results` | 분석 결과 (현황분석·욕구조사·추진방향 A·B·C형) |
| `direction_drafts` | 추진방향 초안·확정·담당자 수정 이력·확정 플래그 |
| `documents` | 생성된 문서 이력 (계획안·결과보고서·출력일·담당자) |

### 운영 원칙

- Claude API 직접 연동 없음 — 분석은 이 프로젝트 세션에서 담당자가 실행
- 시스템은 결과를 구조화해서 저장하고 문서로 출력하는 역할만 담당
- "사업별 만족도 설문" 명칭은 시스템 전체에서 **"현장설문"** 으로 통일

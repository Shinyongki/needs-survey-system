## 사전 작업 — question_labels 테이블 추가

> ⚠️ 3개 기능 모두 이 테이블에 의존하므로, 병렬 개발 전에 먼저 완료해야 함.

### 배경

현재 responses에는 question_no(숫자)만 저장되고 CSV 컬럼명(label)은 버려진다. 문항이 추가·삭제·순서 변경되면 question_no가 달라지므로, 연도 간 비교나 이행 확인에서 "같은 문항"을 식별할 수 없다.

### 테이블 정의

```sql
CREATE TABLE IF NOT EXISTS question_labels (
  id           INTEGER PRIMARY KEY AUTOINCREMENT,
  session_id   INTEGER REFERENCES survey_sessions(session_id),
  question_no  INTEGER,
  label        TEXT,
  UNIQUE(session_id, question_no)
);
```

### CSV 업로드 시 자동 저장 (responses.js 수정)

```javascript
const questionColumns = headers.filter(h => !skipColumns.includes(h));

const existingLabels = db.prepare(
  'SELECT COUNT(*) as cnt FROM question_labels WHERE session_id = ?'
).get(sessionId);

if (existingLabels.cnt === 0) {
  const insertLabel = db.prepare(
    'INSERT INTO question_labels (session_id, question_no, label) VALUES (?, ?, ?)'
  );
  questionColumns.forEach((label, index) => {
    insertLabel.run(sessionId, index + 1, label);
  });
}
```

### 문항 목록 API

**GET `/api/questions?session_id=1`**

```json
{
  "session_id": 1,
  "questions": [
    { "question_no": 1, "label": "기본-0_연령대" },
    { "question_no": 36, "label": "B1-1_교육주제(복수)" }
  ]
}
```

### 연도 간 동일 문항 매칭

```sql
SELECT ql.question_no, ql.label
FROM question_labels ql
JOIN survey_sessions ss ON ql.session_id = ss.session_id
WHERE ss.survey_code = 'S01' AND ss.year = 2025 AND ql.label LIKE 'B1-1%';
```

문항 순서가 바뀌어도 label 패턴으로 매칭 가능.

## 기능 1 — 시계열 추세 `/trends`

### 목적
같은 설문의 연도별 응답 결과를 비교하여 욕구 변화를 추적한다.

### 핵심: label 기반 연도 간 비교

```
사용자: "S01의 B1-1_교육주제를 2025 vs 2026 비교"
시스템:
  → 2025 S01에서 label LIKE 'B1-1%' → question_no = 34
  → 2026 S01에서 label LIKE 'B1-1%' → question_no = 36
  → 각각의 응답 분포를 조회하여 비교
```

### 화면 레이아웃

```
┌─────────────────────────────────────────────────────────┐
│  [설문 선택▼ S01]   [문항 선택▼ B1-1_교육주제]           │
│                                                         │
│  ┌─ 연도별 비교 ──────────────────────────────────────┐  │
│  │   2025년 (N=112)          2026년 (N=119)           │  │
│  │   ① 퇴원환자 실무  32%    ① 특화·위기사례  28%     │  │
│  │   ② 특화·위기사례  24%    ② 퇴원환자 실무  26%     │  │
│  │   ③ 안전사고 대응  18%    ③ 리더십·슈퍼비전 20%    │  │
│  └────────────────────────────────────────────────────┘  │
│                                                         │
│  ┌─ 추세 차트 (5점 척도 문항일 때) ───────────────────┐  │
│  │   4.5 ┤                         ●                  │  │
│  │   4.0 ┤           ●                                │  │
│  │   3.5 ┤  ●                                         │  │
│  │       └──2024──────2025──────2026──                 │  │
│  └────────────────────────────────────────────────────┘  │
│                                                         │
│  변화 요약: "1순위가 '퇴원환자 실무'에서                  │
│             '특화·위기사례'로 변경 (±6%p)"                │
└─────────────────────────────────────────────────────────┘
```

### 표시 방식

| 문항 유형 | 표시 방식 |
|---|---|
| 5점 척도 | 연도별 평균 점수 라인 차트 + 증감 |
| 단일선택 | 연도별 선택지 비율 비교 테이블 |
| 복수선택 | 연도별 선택지 비율 가로 막대 차트 |
| 주관식 | "텍스트 문항은 시계열 비교 대상이 아닙니다" |

### API

**GET `/api/trends?survey_code=S01&question_label=B1-1_교육주제(복수)&years=2025,2026`**

```json
{
  "survey_code": "S01",
  "question_label": "B1-1_교육주제(복수)",
  "years": [
    {
      "year": 2025, "question_no": 34, "respondent_count": 112,
      "distribution": [{"value":"1","count":36,"ratio":0.32}],
      "mean": null
    },
    {
      "year": 2026, "question_no": 36, "respondent_count": 119,
      "distribution": [{"value":"2","count":33,"ratio":0.28}],
      "mean": null
    }
  ]
}
```

**GET `/api/trends/available`** — 설문·연도·문항 목록

```json
{
  "surveys": [
    {
      "survey_code": "S01", "years": [2025,2026],
      "questions": [
        { "label": "B1-1_교육주제(복수)", "available_years": [2025,2026] },
        { "label": "기본-0_연령대", "available_years": [2026] }
      ]
    }
  ]
}
```

## 기능 2 — 기관 비교 `/compare`

### 목적
기관 개별·시군구·규모별로 응답 패턴을 비교한다.

### 화면 레이아웃

```
┌─────────────────────────────────────────────────────────┐
│  [설문 선택▼ J01]  [회차 선택▼ 2026년 1회차]             │
│  [문항 선택▼ 공통5_종합만족도]  ← label 드롭다운         │
│                                                         │
│  [비교 기준]  ○ 기관 개별  ● 시군구  ○ 규모별            │
│                                                         │
│  시군구      응답수   평균    전체평균 대비               │
│  창원시        28    4.21    +0.15  ▲                   │
│  진주시        15    4.13    +0.07  ▲                   │
│  ─── 전체 ───       4.06                                │
│  거제시         8    3.88    -0.18  ▼                   │
│                                                         │
│  [막대 차트]                                             │
└─────────────────────────────────────────────────────────┘
```

### 비교 기준 3가지

| 기준 | 그룹핑 | 활용 |
|---|---|---|
| 기관 개별 | agency_code별 | 특정 기관 확인 |
| 시군구 | agencies.district별 | 지역 간 차이 |
| 규모별 | agencies.size_category별 | 규모별 패턴 |

### API

**GET `/api/compare?session_id=1&question_label=공통5_종합만족도&group_by=district`**

```json
{
  "session_id": 1, "question_label": "공통5_종합만족도",
  "group_by": "district", "overall_mean": 4.06, "overall_count": 92,
  "groups": [
    { "group_key": "창원시", "respondent_count": 28, "mean": 4.21, "diff_from_overall": 0.15 }
  ]
}
```

### API

**GET `/api/linkage?year=2026&mode=auto`** — 자동 매칭 실시간 계산

**GET `/api/linkage?year=2026&mode=manual`** — 수동 저장 목록

**POST `/api/linkage`** — 수동 연결 저장 (label 기반)

```json
{
  "topic_label": "퇴원환자 서비스 교육",
  "need_session_id": 1,
  "need_label": "B1-1_교육주제(복수)",
  "field_session_id": 3,
  "field_label": "공통5_종합만족도"
}
```

**DELETE `/api/linkage/:linkage_id`** — 수동 연결 삭제

**GET `/api/linkage/:linkage_id/evidence`** — 근거 문장 생성

### linkages 테이블 확장

```sql
ALTER TABLE linkages ADD COLUMN mode TEXT DEFAULT 'manual';
ALTER TABLE linkages ADD COLUMN need_label TEXT;
ALTER TABLE linkages ADD COLUMN field_label TEXT;
```

### status 판정 기준

| mean_score | status | 표시 |
|---|---|---|
| ≥ 4.0 | fulfilled | ✅ 이행 확인 |
| < 4.0 | low_satisfaction | ⚠️ 만족도 낮음 |
| 데이터 없음 | no_data | ❌ 데이터 없음 |

### 차트 라이브러리

Recharts (라인·막대·테이블)

### 데이터 없는 경우

- 데이터 없음: "해당 조건의 응답 데이터가 없습니다"
- 비교 연도 1개: "비교할 다른 연도의 데이터가 필요합니다"
- question_labels 없음: "CSV를 먼저 업로드하면 문항 목록이 자동 생성됩니다"

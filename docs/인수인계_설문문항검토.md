## 1. 프로젝트 핵심 개요

**기관:** (재)경상남도사회서비스원 — 노인맞춤돌봄서비스 광역지원기관  
**목표:** 계획→실행→평가→환류 구조의 설문·문서 체계 설계 및 시스템 구현  
**핵심 논리:** ①문항설계 → ②응답수집 → ③결과분석 → ④근거문장 생성 → ⑤사업운영 근거확보  
**근거 문장 형식:** "○○조사 결과 ○○%가 ○○을 요청하여, ○○을 추진한다."

### 역할 구분
- **이 프로젝트(Claude):** 기획·분석·설계·문서 작성
- **Claude Code:** 실제 시스템 구현 (React + Node.js + SQLite, `C:\Projects\설문시스템`)
- **세션 간 연결:** 번호가 붙은 지시문서(맥락문서)로 Claude Code에 전달

### 주요 용어 주의
- "경영평가" 사용 금지 → "사업 운영 근거 확보"로 대체
- "사업별 만족도 설문" → "현장설문"
- "유공자 표창지원사업" → "유공자 표창" (J16)

### 현장설문 공통 문항 구조 (전 사업 동일)

**도입부 3문항:** 연령대 / 노인맞춤돌봄서비스 경력 / 소속 권역  
**경력 구분:** 1년 미만 · 1~3년 · 3~5년 · 5년 이상 (4구간, 현행 유지)  
**권역 구분:** 북부(거창·함양·합천·산청) / 서부(진주·사천·고성·남해·하동) / 중부(창원·함안·의령·창녕) / 동부(김해·양산·밀양·통영·거제)

**공통 4문항:** 5개 그룹으로 운영 (교육형 / 점검·협의형 / 전문지원형 / 특화지원서비스 전용 / 자원·물품형 / 홍보·행정형)  
- 공통-1·2: 특화 문항의 전제 조건  
- 공통-3: 결과 종합 (IPA 분석 중요도 대리지표)  
- 공통-4: 종합 만족도 (전 사업 고정 — 사업 간 비교 유일 지표)

**특화 5문항:** 사업 고유 경험·품질 측정 (마지막은 주관식 "바라는 점")

## 4. 다음 세션 작업 범위

### 목적
작성된 상반기·하반기 통합설문(S01·S02)과 현장설문 16종(J01·J03~J17)의 **문항 내용을 검토**한다.

### 검토 관점
1. **문항 논리 일관성:** 공통-1·2가 특화 문항의 전제 조건으로 기능하는가
2. **중복 여부:** 공통 문항과 특화 문항 간 내용 겹침
3. **분석 패턴과의 연결:** 사업별 분석 패턴(교육 품질 3요소, 프로세스 단계, 응대 품질 이중 구조 등)과 문항이 실제로 연결되는가
4. **현장 실현 가능성:** 응답자(전담사회복지사·생활지원사) 입장에서 이해 가능한 문항인가
5. **통합설문:** 현황분석 문항(현○○)과 욕구조사 문항의 구분이 명확한가

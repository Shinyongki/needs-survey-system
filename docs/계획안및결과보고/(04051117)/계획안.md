## 1. 추진 개요

### 1-1. 목적

노인맞춤돌봄서비스 광역지원기관 사업의 근거 기반 운영 강화를 위해 현황분석·욕구조사 체계를 신규 도입하고, 계획안 및 결과보고서에 데이터 기반 추진 근거를 확보함을 목적으로 한다.

### 1-2. 배경

- 사업 운영 근거 확보 및 실효성 강화를 위해 **현황분석·욕구조사** 체계 신규 도입
- 기존 계획안 구조에 데이터 기반 추진방향 근거 반영

| 구분 | 기존 | 변경 |
|---|---|---|
| 계획안 구조 | 개요 → 추진방향 → 세부계획 → 향후계획 | **현황분석 → 욕구조사 →** 추진방향 → 세부계획 → 향후계획 |
| 추진방향 근거 | 담당자 판단 중심 | **데이터 + 종사자 욕구** 기반 |

### 1-3. 필요성

- **현황분석**: 객관적 데이터로 현재 상황 파악 → "왜 이 사업이 필요한가" 증명
- **욕구조사**: 종사자 수요 직접 파악 → "무엇을 어떻게 해야 하는가" 방향 근거
- **사업 운영 근거 확보**: 계획안·결과보고서에 추진 근거 명시 필요

> 근거 문장 형식: **"○○조사 결과 ○○%가 ○○을 요청하여, ○○을 추진한다."**

| 사업명 | 노인맞춤돌봄서비스 현황분석·욕구조사 체계 도입 |

## 2. 조사·분석 운영 체계

### 2-1. 현황분석

#### 개념 및 원칙

- **정의**: 사업 기획 전 현재 상황을 객관적 데이터로 파악하는 과정
- **핵심 원칙**: 단순 수치 나열이 아닌, **수치 → 시사점** 형태로 분석
- **데이터 출처**: 전년도 결과보고서, 모니터링 기록, 모인우리 기관 관리 데이터, 상·하반기 통합설문 현황분석 결과, 현장메모
- **시기**: 매년 1~2월 (계획안 수립 전)

#### 현황분석 항목

| 분석 영역 | 주요 항목 | 활용 사업 |
|---|---|---|
| 수행기관 현황 | 기관 수, 신규기관 수, 운영 연차 분포 | 모니터링·점검, 전체 |
| 종사자 현황 | 전담사회복지사·생활지원사 수, 신규 입사·이직 현황 | 역량강화교육·심리지원교육, 종사자 지원사업 |
| 서비스 제공 현황 | 대상자 수, 서비스 유형별 현황 | 특화지원·퇴원환자, ICT 지원·나눔사업·캠페인 |
| 전년도 점검 결과 | 지적사항 유형·빈도, 반복 지적 기관 수 | 모니터링·점검, 컨설팅 지원 |
| 전년도 교육 결과 | 참여율, 주제별 만족도 | 역량강화교육·심리지원교육 |
| 상담·연계 실적 | 컨설팅·노무 상담 건수·유형, 특화지원·퇴원환자 서비스 연계 건수 | 컨설팅·특화지원·퇴원환자·노무 상담 |
| 종사자 안전·심리 | 우울·불안·직무스트레스 위험군 비율, 감정노동 관련 스트레스 수준 (종사자 심리진단 결과보고서) | 심리지원교육, 종사자 지원사업 |
| 현장메모 | 사업 진행 중 현장 변동사항·특이사항 | 전체 |

#### 분석 방법

1. **데이터 수집** — 시스템·보고서·기록 등에서 항목별 수치 추출
2. **수치 분석** — 전년 대비 증감, 유형별 비율, 권역별 편차
3. **시사점 도출** — 수치 → 사업 강화 방향 연결
4. **계획안 반영** — 추진방향의 '현황 근거' 문장으로 명시

> ※ 2026년(첫해): 모인우리 기관 관리 데이터 및 전년도 결과보고서 활용  
> ※ 2027년 이후: 상·하반기 통합설문 현황분석 결과 + 현장메모 추가 투입

### 2-2. 욕구조사

#### 개념 및 원칙

- **정의**: 종사자가 광역지원기관에 어떤 지원을 원하는지 직접 파악
- **조사 대상**: 전담사회복지사, 생활지원사
- **핵심 원칙**: 응답 비율(%)을 추진방향 근거 문장에 명시
- **결과 활용**: 당해 연도 사업 조정 + 다음 연도 계획안 추진방향 근거

**특화 문항**

특화 문항은 각 사업 고유의 경험·품질을 측정하며, 욕구조사는 통합설문에 위임하고 이번 회차 경험 평가에 집중한다. 마지막 문항은 주관식(바라는 점)으로 구성된다.

특화지원서비스의 경우 광역 지원 기능 4차원을 측정한다.

| 코드 | 측정 차원 |
|---|---|
| ① | 대상자 선별·발굴 과정 광역 안내·지원 |
| ② | 서비스 계획(개별·집단·치료) 수립 자문 |
| ③ | 어려운 사례 진행 중 슈퍼비전 |
| ④ | 외부 전문기관(치료·상담) 연계 정보 제공 |
| ⑤ | 바라는 점 (주관식) |

**기타 (직접 입력) 응답 설계**

선택형 문항 중 주제·분야·방식·이유·어려운점 등 응답자의 경험이 제시된 선택지 외로 나타날 수 있는 문항에는 "기타 (직접 입력)" 항목을 추가하였다. 현장설문에서는 역량강화교육·심리지원교육(생활지원사) 교육주제, 컨설팅 지원 분야, 노무 상담 지원 유형, 안심 돌봄 캠페인 홍보 채널 5개 문항에 적용된다.

### 2-5. 분석 운영

#### 분석 단위

모든 설문 결과는 아래 4개 단위로 분석한다. 기관별 분석은 응답자 익명성 보장을 위해 실시하지 않는다.

| 분석 단위 | 내용 |
|---|---|
| 전체 | 경남 전체 응답 종합 |
| 권역별 | 북부(거창·함양·합천·산청) / 서부(진주·사천·고성·남해·하동) / 중부(창원·함안·의령·창녕) / 동부(김해·양산·밀양·통영·거제) |
| 연령대별 | 20대·30대·40대·50대·60대 이상 |
| 경력별 | 1년 미만·1~3년·3~5년·5년 이상 |

#### 기본 분석 (2026년~)

| 분석 | 내용 | 활용 |
|---|---|---|
| 기술통계 | 문항별 빈도·비율·평균 | 근거 문장 핵심 재료 |
| 만족도지수(CSI) | 긍정비율(4+5) 100점 환산·사업 간 순위 | 사업별 성과 비교 |
| 균형 프로파일 | 다차원 척도를 레이더 차트로 시각화 | 사업별 강·약점 파악 |
| 갭(Gap) 분석 | 항목 간 점수 차이, 단계별 병목 진단 | 개선 우선순위 설정 |
| IPA 분석 | 중요도(공통-3 대리) × 만족도 4분면 | 자원 집중 방향 결정 |
| 욕구 순위 | 선택형 문항 항목별 빈도·비율, Top 3 | 수요 우선순위 파악 |
| 텍스트 분석 | 주관식 키워드 추출, 긍정·개선요구 분류 | 정성적 수요 파악 |
| 추진 근거 생성 | "○○% 요청 → ○○ 추진" | 계획안·결과보고서 근거 문장 |

#### 심화 분석

| 분석 | 내용 | 가능 시점 |
|---|---|---|
| 교차 분석 | 권역×경력, 권역×연령대 등 변수 2개 이상 조합 | 2026년~ |
| 이행 확인 | 상반기 욕구 → 현장설문 만족도 연결 | 2026년~ |
| 연도 간 변화 추세 | 동일 문항 전년도 대비 변화 비교 | 2027년~ |
| 상관 분석 | 특화 문항 간 상관, 종합만족도 영향 요인 분석 | 2026년~ |

#### 기타 응답 처리

구글폼에서 "기타 (직접 입력)"을 선택한 응답은 숫자 코드가 아닌 자유 텍스트로 저장된다. 분석 시 아래 원칙을 따른다.

- 기타 응답이 **5% 이상**이면 내용 요약 필수
- 기타 응답이 **3건 이상**이면 반복 키워드 추출
- 기타 내용에 새 카테고리가 보이면 별도 표시 — 다음 연도 선택지 보완 근거로 활용

#### 연도별 분석 확대 계획

> **[그림 1] 연도별 분석 확대 계획**

- 2026년: 기술통계, CSI, 갭·IPA·균형프로파일, 추진 근거 생성, 권역·연령대·경력별 비교, 이행 확인, 교차 분석, 상관 분석
- 2027년: 연도 간 변화 추세(2026 vs 2027), 이행 확인 누적
- 2028년 이후: 다년도 추세 — 3개년 이상 시계열 분석

#### 이행 확인 구조

> **[그림 2] 이행 확인 구조**

- 상반기 통합설문 욕구 파악 → 당해 연도 사업 실행에 직접 반영
- 현장설문 만족도 측정 → 욕구 반영 여부(이행 확인) 수행
- 이행 확인 결과: 결과보고서 수록 + 차기 계획안 추진방향 근거로 활용
- 매년 반복되며, 연수가 쌓일수록 근거가 누적·강화됨

### 2-6. 분석지시어 체계

분석지시어는 설문 데이터(CSV)를 Claude 등 AI 분석 도구에 입력할 때 사용하는 표준화된 프롬프트 모음이다. 담당자가 지시어를 복사·붙여넣기만으로 일관된 분석 결과를 도출할 수 있도록 설계되었다.

#### 분석지시어 3종 구성

| 파일명 | 대상 설문 | 주요 내용 |
|---|---|---|
| 상반기통합설문_분석지시어모음 v4 | 상반기 통합설문 (56문항) | 유형1~9 (현황집계·욕구우선순위·교차·근거문장·시각화·섹션집중·보고서·비교) |
| 하반기통합설문_분석지시어모음 v4 | 하반기 통합설문 (59문항) | 상반기 동일 구조 + 하반기 고유 항목 (연간 결산, 마무리 종합의견, 상하반기 비교) |
| 현장설문_사업별분석지시어_15종 v4 | 현장설문 15종 | 사업별 M1~M8 모델 (기술통계·CSI·균형프로파일·갭·IPA·욕구순위·텍스트·근거문장) |

#### 분석지시어 설계 원칙

- **재현성**: 동일 지시어 사용 시 일관된 분석 결과 도출
- **모듈화**: 필요한 분석 유형만 선택하여 사용 가능
- **기타 처리 내장**: 기타 응답의 CSV 저장 방식과 처리 방법 명시
  - 단일선택: 숫자 코드 외 텍스트 응답 = 기타로 분류
  - 복수선택: 세미콜론 파싱 시 숫자·텍스트 항목 분리
- **근거문장 자동화**: 분석 수치를 추진방향 근거 문장 형식으로 자동 생성

#### 현장설문 사업별 분석 패턴

각 사업의 성격에 따라 특화된 분석 패턴을 적용한다.

| 패턴 | 해당 사업 | 핵심 분석 |
|---|---|---|
| 교육 품질 3요소 평가형 | 역량강화교육(전담사회복지사·생활지원사), 심리지원교육(전담사회복지사·생활지원사) | 주제관련성·전문성·운영방식 3요소 균형 + 현장 적용 가능성 |
| 프로세스 단계 평가형 | 수행기관 모니터링·점검 | 점검→피드백→개선지원 3단계 갭 분석 + 병목 지점 진단 |
| 운영 3요소 균형형 | 실무협의회 | 주제 관련성·참여 반영·방식 편의 + 사례 공유 충분성 |
| 응대 품질 이중 구조형 | 컨설팅 지원, 노무 상담 지원 | 응대 신속성 vs 문제 해결력 갭 + 이용 유형 분석 |
| 광역 지원 4차원 평가형 | 특화지원서비스 | 선별발굴·계획수립·슈퍼비전·연계정보 4차원 IPA |

| 효과 확인→수요 전환형 | 종사자 지원사업 | 올해 도움된 지원 vs 내년 필요 지원 수요 전환 분석 |

|---|---|---|
| 1단계 기반 인프라 | 기관 관리, CSV 업로드, 응답 현황, 신청자 명단, 응답 번호 관리 | ✅ 완료 |

사업 종료 → 구글폼 설문 실시 → CSV 내보내기 → 시스템 업로드 → 분석 API 처리 → 분석지시어로 결과 활용 → 계획안·결과보고서 작성의 흐름으로 운영된다.

분석 작업은 시스템에서 생성된 집계 텍스트를 복사하여 Claude 등 AI 분석 도구에 분석지시어와 함께 입력하는 방식으로 수행하며, 별도 API 연동 없이 운영 가능하다.

## 3. 추진방향 연결 구조

### 3-1. 추진 근거 유형 및 형식

추진 근거란 계획안·결과보고서에 기재되는 데이터 기반 의사결정 문장이다. "이 사업을 왜 이렇게 운영하는가"를 담당자의 판단이 아닌 조사 결과(수치)로 직접 증명하며, 출처와 종류에 따라 아래 5가지 유형으로 구분한다.

> **[그림 3] 근거 문장 구조**

| 유형 | 근거 출처 | 예시 문장 |
|---|---|---|
| 욕구조사 | 통합설문 응답 비율 | "[설문명] 결과 [응답비율]%가 [요청사항]을 요청하여, [추진내용]을 추진한다." |
| 현황분석 | 행정 데이터·모니터링 기록 | "전년도 [분석항목] 결과 [현황내용]이 확인되어, [추진내용]을 강화한다." |
| 변화 추세 | 전년도 비교 (2027년~) | "전년도 대비 [요청사항] 비율이 [증감폭]%p 변화하여, [추진내용]을 확대한다." |

## 4. 기대효과

- **사업 운영 근거 확보**: 근거 문장 명시로 "근거 기반 사업 기획" 요건 충족
- **사업 실효성 향상**: 현장 수요 기반 교육 주제·지원 방식 선택
- **환류 사이클 완성**: 욕구 요청 → 반영 → 만족도 확인 → 다음 연도 반영
- **종사자 체감도 향상**: 욕구 결과가 실제 사업에 반영됨을 확인 가능
- **현장 밀착 관리**: 현장메모로 설문 외 변동사항 실시간 수집
- **다각도 분석**: 권역·연령대·경력별 비교 + 연도 간 추세 추적
- **분석 표준화**: 분석지시어를 통한 일관된 분석 결과 도출
- **운영 효율화**: 전용 시스템으로 CSV 업로드~분석 결과 출력 자동화

# 지시문 02: DB 스키마 확장 (권역·연령대·경력 컬럼 추가)

> Wave 1 — 독립 실행 가능  
> 선행 조건: 없음  
> 프로젝트 경로: C:\Projects\설문시스템

---

## 작업 목적

분석 단위가 기관별 → 전체·권역·연령대·경력별로 변경됨에 따라,  
responses 테이블에 3개 메타 컬럼을 추가하고 권역 코드 정의를 상수로 관리한다.

---

## 변경 내용

### responses 테이블에 추가할 컬럼

| 컬럼명 | 타입 | 허용값 | 설명 |
|---|---|---|---|
| region | TEXT | '북부'·'서부'·'중부'·'동부'·NULL | 소속 권역 (도입-3 응답값) |
| age_group | TEXT | '20대'·'30대'·'40대'·'50대'·'60대이상'·NULL | 연령대 (도입-1 응답값) |
| career | TEXT | '1년미만'·'1~3년'·'3~5년'·'5년이상'·NULL | 경력 (도입-2 응답값) |

> NULL 허용: 기존 데이터 및 파싱 실패 시 NULL 저장

### 권역 구분 정의 (백엔드 상수로 관리)

```
북부: 거창군, 함양군, 합천군, 산청군
서부: 진주시, 사천시, 고성군, 남해군, 하동군
중부: 창원시, 함안군, 의령군, 창녕군
동부: 김해시, 양산시, 밀양시, 통영시, 거제시
```

## 실행 절차

### 1단계: 현재 responses 테이블 구조 확인

```sql
PRAGMA table_info(responses);
```

### 2단계: 컬럼 추가

```sql
ALTER TABLE responses ADD COLUMN region TEXT;
ALTER TABLE responses ADD COLUMN age_group TEXT;
ALTER TABLE responses ADD COLUMN career TEXT;
```

### 3단계: 권역 상수 파일 생성

`backend/constants/regions.js` 파일 생성:

```javascript
const REGIONS = {
  북부: ['거창군', '함양군', '합천군', '산청군'],
  서부: ['진주시', '사천시', '고성군', '남해군', '하동군'],
  중부: ['창원시', '함안군', '의령군', '창녕군'],
  동부: ['김해시', '양산시', '밀양시', '통영시', '거제시'],
};

// 설문 도입-3 응답 선택지 → region 코드 매핑
const REGION_LABELS = {
  '북부(거창·함양·합천·산청)': '북부',
  '서부(진주·사천·고성·남해·하동)': '서부',
  '중부(창원·함안·의령·창녕)': '중부',
  '동부(김해·양산·밀양·통영·거제)': '동부',
};

// 연령대 선택지 → age_group 코드 매핑
const AGE_GROUP_LABELS = {
  '20대': '20대',
  '30대': '30대',
  '40대': '40대',
  '50대': '50대',

// 경력 선택지 → career 코드 매핑
const CAREER_LABELS = {
  '1년 미만': '1년미만',
  '1~3년': '1~3년',
```

### 4단계: 완료 확인

```sql
PRAGMA table_info(responses);
```

기대 결과: region, age_group, career 컬럼이 목록에 존재할 것

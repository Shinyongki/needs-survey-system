## 1. 프로젝트 개요

경상남도사회서비스원 광역지원기관이 59개 수행기관의 설문 데이터를 수집·저장·분석하는 시스템.

**핵심 목적 3가지:**
1. 시계열 누적 — 연도별·회차별 욕구 변화 추적
2. 기관 간 비교 — 지역·규모·유형별 응답 패턴 비교
3. 이행 확인 — 상반기 욕구 요청 → 현장설문 만족도 자동 연결

## 4. DB 스키마

### 4-1. agencies (수행기관)

```sql
CREATE TABLE IF NOT EXISTS agencies (
  agency_code     TEXT PRIMARY KEY,        -- A48310001 형식
  agency_name     TEXT NOT NULL,
  district        TEXT NOT NULL,           -- 지자체 (거제시, 창원시 등)
  facility_type   TEXT,                    -- 시설유형
  sw_count        INTEGER,                 -- 전담사회복지사 배정 수
  ls_count        INTEGER,                 -- 생활지원사 배정 수
  client_count    INTEGER,                 -- 대상자 배정 수
  size_category   TEXT GENERATED ALWAYS AS (
    CASE
      WHEN client_count >= 1000 THEN '대형'
      WHEN client_count >= 700  THEN '중형'
      ELSE '소형'
    END
  ) STORED,
  created_at      TEXT DEFAULT (datetime('now', 'localtime'))
);
```

### 4-2. survey_sessions (설문 회차)

```sql
CREATE TABLE IF NOT EXISTS survey_sessions (
  session_id      INTEGER PRIMARY KEY AUTOINCREMENT,
  survey_code     TEXT NOT NULL,   -- S01, S02, J01, J03~J17
  survey_name     TEXT NOT NULL,   -- 상반기통합설문, 현장설문_실무협의회 등
  year            INTEGER NOT NULL,
  half            TEXT,            -- 'first'(상반기) | 'second'(하반기) | NULL(현장설문)
  round_no        INTEGER DEFAULT 1, -- 회차 (현장설문 다회차 대응)
  target_role     TEXT NOT NULL,   -- 'sw'(전담사회복지사) | 'ls'(생활지원사)
  status          TEXT DEFAULT 'open', -- open | closed
  created_at      TEXT DEFAULT (datetime('now', 'localtime'))
);
```

### 4-3. responses (응답 데이터 — 문항별 저장)

```sql
CREATE TABLE IF NOT EXISTS responses (
  response_id     INTEGER PRIMARY KEY AUTOINCREMENT,
  session_id      INTEGER NOT NULL REFERENCES survey_sessions(session_id),
  agency_code     TEXT NOT NULL REFERENCES agencies(agency_code),
  respondent_role TEXT NOT NULL,   -- 'sw' | 'ls'
  question_no     INTEGER NOT NULL,
  answer_value    TEXT,            -- 단일·복수 모두 TEXT로 저장 (복수선택은 세미콜론 구분)
  answer_type     TEXT,            -- 'likert' | 'single' | 'multi' | 'text'
  submitted_at    TEXT DEFAULT (datetime('now', 'localtime'))
);
```

### 4-4. linkages (이행 확인 연결)

```sql
CREATE TABLE IF NOT EXISTS linkages (
  linkage_id      INTEGER PRIMARY KEY AUTOINCREMENT,
  need_session_id INTEGER NOT NULL REFERENCES survey_sessions(session_id), -- 상반기 욕구 회차
  field_session_id INTEGER NOT NULL REFERENCES survey_sessions(session_id), -- 현장설문 회차
  topic_label     TEXT NOT NULL,   -- 연결 항목명 (예: '심리지원 프로그램')
  need_ratio      REAL,            -- 욕구 요청 비율 (%)
  satisfaction    REAL,            -- 현장설문 만족도 점수
  created_at      TEXT DEFAULT (datetime('now', 'localtime'))
);
```

### 4-5. documents (문서 이력)

```sql
CREATE TABLE IF NOT EXISTS documents (
  doc_id          INTEGER PRIMARY KEY AUTOINCREMENT,
  doc_type        TEXT NOT NULL,   -- '계획안' | '결과보고서'
  year            INTEGER NOT NULL,
  content_json    TEXT,            -- 현황분석·욕구조사·추진방향 JSON
  is_confirmed    INTEGER DEFAULT 0,
  created_at      TEXT DEFAULT (datetime('now', 'localtime')),
  confirmed_at    TEXT
);
```

## 5. API 인터페이스 (트랙 B·C 공유)

### 기관 관리
```
GET    /api/agencies              ← 전체 기관 목록
GET    /api/agencies/:code        ← 기관 단건
PUT    /api/agencies/:code        ← 기관 정보 수정
```

### 설문 회차
```
GET    /api/sessions              ← 회차 목록 (연도·설문코드 필터)
POST   /api/sessions              ← 회차 등록
PATCH  /api/sessions/:id/close   ← 회차 마감
```

### CSV 업로드
```
POST   /api/responses/upload      ← CSV 파싱 후 DB 저장
  body: { session_id, csv_data }
  반환: { inserted, skipped, errors }
```

### 응답 현황
```
GET    /api/dashboard/status      ← 기관×설문 응답 현황 매트릭스
  query: year, session_id(optional)
  반환: { agencies[], sessions[], matrix{} }
```

## 6. CSV 형식 (구글폼 연동)

구글폼 드롭다운에서 기관을 선택하면 아래 형식으로 CSV가 생성됨:

```
타임스탬프, 기관선택, Q1, Q2, Q3, ..., Q10
2026-04-01, A48310001_거제노인통합지원센터, 4, 3, 5, ...
```

파싱 규칙:
- `기관선택` 컬럼에서 `_` 앞부분을 agency_code로 추출
- Q1~Q10(또는 Q55)을 question_no 1~N으로 순서대로 저장
- 복수선택 항목은 세미콜론(`;`) 구분으로 저장
